# 보안 정책

K-Game 저장소는 공개 저장소 기준으로 운영됩니다.  
따라서 코드가 동작하는 것만큼이나, 비밀정보와 운영 정보가 노출되지 않는 것이 중요합니다.

## 비밀정보 처리 원칙

- `.env`, API 키, DB 비밀번호, 토큰, 세션 값은 저장소에 올리지 않습니다.
- 운영 비밀은 로컬 전용 파일이나 AWS Secrets Manager, SSM Parameter Store 같은 외부 비밀 저장소를 사용합니다.
- 예제만 필요한 경우에는 실제 값이 없는 `.env.example`만 남깁니다.
- 로그와 스크린샷에 민감정보가 찍히지 않도록 마스킹합니다.

## 취약점 신고

- 공개 이슈에는 취약점의 세부 재현 절차나 민감한 정보를 적지 않습니다.
- 취약점은 가능한 경우 GitHub Security Advisory 같은 비공개 경로를 사용합니다.
- 비공개 경로를 쓸 수 없는 경우에는 저장소 관리자에게 직접 비공개로 연락합니다.

## 운영 원칙

- 운영용 계정과 개발용 계정은 분리합니다.
- 최소 권한 원칙을 적용하고, 필요한 권한만 허용합니다.
- RDS 같은 내부 자원은 가능하면 비공개 네트워크에 둡니다.
- 외부에 노출되는 API는 입력 검증, 인증, 에러 처리, 로깅 마스킹을 기본으로 둡니다.
- 배포 전에 보안 체크리스트를 확인하고, 공개 가능한 상태인지 다시 점검합니다.

## 신고 시 포함하면 좋은 정보

- 취약점이 발생한 화면 또는 API
- 영향 범위
- 재현 절차
- 기대 동작과 실제 동작
- 관련 로그 또는 스크린샷
