# 문제 해결

## 프론트가 API와 연결되지 않을 때

- `apps/web/.env.example`의 API 주소를 확인합니다.
- API가 실제 포트에서 동작하는지 확인합니다.
- CORS origin이 프론트 주소와 일치하는지 확인합니다.

## 로그인 세션이 유지되지 않을 때

- 브라우저 쿠키 차단 여부를 확인합니다.
- 응답 헤더에 `Set-Cookie`가 내려오는지 확인합니다.
- `Secure`와 `SameSite` 설정이 현재 환경과 맞는지 확인합니다.

## Lambda 호출이 실패할 때

- 함수 이름과 리전을 확인합니다.
- EC2 역할에 호출 권한이 있는지 확인합니다.
- CloudWatch 로그에서 에러 메시지를 확인합니다.

## DB 연결이 실패할 때

- `services/api/.env.example` 기준으로 DB 값을 확인합니다.
- RDS 보안 그룹이 API 보안 그룹만 허용하는지 확인합니다.
- 마이그레이션이 먼저 적용되었는지 확인합니다.
