# 저장소 가이드

## 최상위 구조

- `apps`: 사용자에게 보이는 화면
- `services`: API와 Lambda 작업 단위
- `infra`: 배포와 AWS 선언
- `docs`: 설명 문서
- `.github`: CI와 템플릿

## 읽는 순서

1. `README.md`
2. `docs/project-overview.md`
3. `docs/architecture.md`
4. `docs/deployment.md`
5. `docs/security-checklist.md`

## 공개 기준

- 예시 파일만 Git에 둡니다.
- 비밀값은 문서에 직접 쓰지 않습니다.
- 불필요한 산출물과 개인용 임시 파일은 제외합니다.
