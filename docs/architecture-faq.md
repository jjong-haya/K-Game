# 아키텍처 FAQ

## 왜 프론트와 API를 분리했나요?

화면 렌더링과 데이터 처리를 나누면 배포와 장애 범위가 줄어듭니다.  
프론트는 정적 자산 배포에 집중하고, API는 비즈니스 로직과 인증에 집중합니다.

## 왜 S3 + CloudFront인가요?

정적 자산을 빠르게 배포하고, HTTPS와 캐시 정책을 한 곳에서 관리하기 좋기 때문입니다.  
S3를 직접 공개하는 방식보다 운영 설명이 더 깔끔합니다.

## 왜 ALB + EC2인가요?

EC2 위에서 Express API를 안정적으로 운영하면서, ALB로 트래픽 분산과 헬스체크를 분리하기 쉽기 때문입니다.

## 왜 RDS를 private subnet에 두나요?

데이터베이스는 외부 직접 접근이 필요하지 않기 때문입니다.  
API만 접근하도록 제한하면 공격면을 줄일 수 있습니다.

## 왜 Lambda를 별도로 두나요?

AI 응답 생성과 판정은 요청 특성이 다르기 때문입니다.  
필요할 때만 확장하고, 로직 책임도 분리할 수 있습니다.

## 어떤 보안이 핵심인가요?

- 세션 쿠키는 `httpOnly`, `Secure`, `SameSite` 기준으로 다룹니다.
- RDS와 EC2는 최소 권한 SG로 묶습니다.
- 비밀값은 코드와 문서에 직접 쓰지 않습니다.
