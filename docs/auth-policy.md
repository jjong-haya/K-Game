# 인증 정책

## 목표

브라우저 저장소에 민감한 토큰을 두지 않고, 서버 발급 세션 쿠키 중심으로 인증합니다.

## 흐름

1. 게스트 또는 소셜 로그인 요청을 보냅니다.
2. `services/api`가 사용자 정보를 검증합니다.
3. 서버가 세션 쿠키를 설정합니다.
4. 이후 요청은 쿠키를 기준으로 인증합니다.

## 원칙

- 클라이언트는 토큰을 장기 저장하지 않습니다.
- 쿠키는 `httpOnly`, `Secure`, `SameSite` 기준을 따릅니다.
- 로그아웃 시 세션 쿠키를 제거합니다.
- 계정 삭제 시 관련 개인 정보는 서버에서 정리합니다.

## 공개 저장소 주의점

- 비밀 키나 토큰을 문서에 직접 쓰지 않습니다.
- 예시 값은 반드시 예시라고 표시합니다.
- 운영 값은 `.env.example`과 Terraform 예시 파일로만 설명합니다.
