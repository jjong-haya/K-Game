# 공개 전 체크리스트

이 문서는 GitHub 공개 직전과 발표 직전에 최종 점검할 항목을 정리한 문서입니다.

## 공개 전

- [ ] 실제 `.env`가 Git에 포함되지 않았는지 확인
- [ ] 로그, zip, build 산출물이 저장소에 남아 있지 않은지 확인
- [ ] 문서에 로컬 절대 경로가 없는지 확인
- [ ] 개인 연락처, 개인 이메일, 개인 식별 정보가 남아 있지 않은지 확인
- [ ] `README.md`에 GitHub Actions 배지와 아키텍처 다이어그램이 있는지 확인
- [ ] `docs/evaluator-guide.md`가 먼저 읽을 문서로 안내되는지 확인
- [ ] `docs/release-checklist.md`가 최신 상태인지 확인
- [ ] `npm run test:web`, `npm run test:api`, `npm run test:lambdas`, `npm run check:infra` 명령이 문서와 일치하는지 확인

## 발표 전

- [ ] 1분 소개를 말할 수 있는지 확인
- [ ] 폴더 구조를 설명할 수 있는지 확인
- [ ] 프론트, API, Lambda, DB의 역할 분리를 설명할 수 있는지 확인
- [ ] 왜 S3와 CloudFront를 썼는지 설명할 수 있는지 확인
- [ ] 왜 EC2와 RDS를 분리했는지 설명할 수 있는지 확인
- [ ] 인증이 왜 세션 쿠키 기반인지 설명할 수 있는지 확인
- [ ] 장애 시 롤백 경로를 말할 수 있는지 확인

## 기술 점검

- [ ] 프론트 빌드가 되는지 확인
- [ ] API 테스트가 통과하는지 확인
- [ ] Lambda 테스트가 통과하는지 확인
- [ ] Terraform 검증이 되는지 확인
- [ ] 배포 스크립트 경로가 실제 구조와 일치하는지 확인
- [ ] README와 docs 링크가 끊기지 않는지 확인

## 제출 직전 한 줄 요약

이 저장소는 코드, 문서, 배포, 보안을 같이 정리한 GitHub 공개용 제출물이어야 합니다.
